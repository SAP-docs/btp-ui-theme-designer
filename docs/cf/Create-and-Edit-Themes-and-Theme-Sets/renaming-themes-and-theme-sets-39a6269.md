# Renaming Themes and Theme Sets

You can rename themes and theme sets to change their properties.



## Prerequisites

-   You are on the welcome page as described in [Managing Themes and Theme Sets on the Welcome Page](managing-themes-and-theme-sets-on-the-welcome-page-e358508.md).
-   You have created a theme or theme set as described in [Creating New Themes and Theme Sets](creating-new-themes-and-theme-sets-72c730b.md).



## Procedure

1.  Select the theme or theme set you want to rename.
2.  Choose *Rename*.
3.  Change the properties of the theme as described in [Changing Theme Properties](Customizing-Themes-in-the-Editor/changing-theme-properties-9766329.md) or the properties of the theme set as described in [Customizing Theme Sets](customizing-theme-sets-ca5d6be.md).
4.  Confirm with *Rename*.

> ### Note:  
> When you rename a published theme, the change isn't reflected in the SAP Build Work Zone theme manager. End users who had selected this theme will experience a theme fallback as described in [Theme Fallback](../Advanced-Information/theme-fallback-bfe0aa3.md).
> 
> To avoid disrupting end users, consider this alternative approach:
> 
> 1.  Duplicate the old theme to the new name as described in [Duplicating Themes and Theme Sets](duplicating-themes-and-theme-sets-ec77044.md).
> 2.  Move all users to the new theme.
> 3.  Delete the theme with the old name as described in [Deleting Themes and Theme Sets](deleting-themes-and-theme-sets-0729728.md)



## Results

The theme or theme set is renamed and its properties are updated.



## Next steps

Publish the renamed theme or theme set to make it available for your applications as described in [Publishing Themes and Theme Sets](publishing-themes-and-theme-sets-f4889a4.md).

> ### Note:  
> You can change theme properties \(except the Theme ID\) in the theme menu of the editor under *Theme* \> *Properties*. For more information, see [Changing Theme Properties](Customizing-Themes-in-the-Editor/changing-theme-properties-9766329.md).
