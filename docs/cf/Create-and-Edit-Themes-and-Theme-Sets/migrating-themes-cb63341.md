# Migrating Themes

Here's how you can migrate custom themes.



## Prerequisites

-   You are on the welcome page as described in [Managing Themes and Theme Sets on the Welcome Page](managing-themes-and-theme-sets-on-the-welcome-page-e358508.md).
-   You have created a theme or theme set as described in [Create and Edit Themes and Theme Sets](create-and-edit-themes-and-theme-sets-0d2d662.md).



<a name="loiocb63341115d444ac8bec8bb44ad63498__steps_hzb_k1n_pl"/>

## Procedure

1.  Select the theme you want to migrate.

2.  Choose *Migrate*.

3.  Provide the *Theme ID* and *Title* of the new theme.

4.  For *Based On*, select the SAP theme you want to use as the base theme for the migrated theme.

5.  Confirm with *Migrate*.




## Results

A new theme is created with the specified *Theme ID* and *Title*. This theme contains the changes from the original custom theme but is based on the SAP theme that you selected.

> ### Note:  
> -   The parameter structure differs between themes. What works in one theme might not work in another. Check and fine-tune the results.
> 
> -   When you migrate a theme based on Belize, only the light part of the theme is copied. You can't copy the contrast part instead.



## Next Steps

Check the migration result and edit your theme as described in [Customizing Themes in the Editor](Customizing-Themes-in-the-Editor/customizing-themes-in-the-editor-aa4b233.md).
