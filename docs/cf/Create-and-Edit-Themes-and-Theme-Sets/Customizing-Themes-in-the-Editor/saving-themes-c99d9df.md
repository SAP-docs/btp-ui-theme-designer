# Saving Themes

You can save changes you made in your theme at any time during its creation and design.



## Prerequisites

You are working in the editor as described in [Customizing Themes in the Editor](customizing-themes-in-the-editor-aa4b233.md).



## Procedure

1.  Open the *Theme* menu.

2.  Select an option that saves the theme:

    -   *Save* to save your changes without publishing them
    -   *Save As* to save a copy of the theme with your changes and new properties. This does not modify the original theme.
    -   *Save & Publish* to save your changes and publish them immediately. See [Publishing Themes and Theme Sets](../publishing-themes-and-theme-sets-f4889a4.md) for more information on publishing.

3.  Enter the properties for your theme. See [Changing Theme Properties](changing-theme-properties-9766329.md) for a list of available properties.

4.  To confirm, select *Save*.

    > ### Note:  
    > If you have previously saved the theme, the *Save* and *Save & Publish* options will execute immediately without reopening the properties dialog.




## Results

Your changes are saved. You can revisit them later.

If you select *Save As*, your original theme is not modified. Instead, it saves the changes to a new theme.

If you select *Save & Publish*, the theme with your changes is applied immediately to SAP Build Work Zone sites and applications.



## Next Steps

You can continue to edit your theme at any time, as described in [Customizing Themes in the Editor](customizing-themes-in-the-editor-aa4b233.md).

If you select *Save & Publish*, your administrators can directly enable the theme for end user selection in the SAP Build Work Zone theme manager. If you do not publish immediately, you can publish your changes later. See [Publishing Themes and Theme Sets](../publishing-themes-and-theme-sets-f4889a4.md).
